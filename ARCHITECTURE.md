**Key Design Decisions:**

1. **Separate Collections**: Disease and scheme data kept in distinct vector stores
   - **Why**: Enables targeted retrieval, reduces noise
   - **Trade-off**: Slight complexity in hybrid queries (mitigated by LangGraph)

2. **800-char Chunks with 200-char Overlap**
   - **Why**: Preserves context while maintaining reasonable token count
   - **Alternative considered**: 1000/100 - too long, lost granularity

3. **Metadata Tagging**
   - Each chunk tagged with: `doc_type`, `chunk_id`, `source_file`
   - Enables source citation and filtering
